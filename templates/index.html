Architecture Overview
Client-Server pattern using Flask + Browser

The browser (client) sends HTTP JSON requests to the Flask server (backend). All cryptographic operations happen server-side — the client only displays results. SHA-256 handles authentication; AES handles message confidentiality. Encryption and decryption are only accessible after a successful login.

🔏
SHA-256 Password Hashing Practical 9
Used in Register (store hash) and Login (compare hashes)
import hashlib # Produces a fixed 256-bit (64 hex char) digest from any input def sha256(text: str) -> str: return hashlib.sha256(text.encode()).hexdigest() # REGISTER — hash and store (never keep plain password) users_db[username] = sha256(password) # LOGIN — hash what user typed, compare to stored hash if sha256(entered_pass) == users_db[username]: session["user"] = username # ← access granted

AES-128 CBC Encrypt / Decrypt Practical 5
Symmetric encryption — session required
from Crypto.Cipher import AES from Crypto.Util.Padding import pad, unpad import os, base64, hashlib def aes_encrypt(plaintext, key_str): key = hashlib.sha256(key_str.encode()).digest()[:16] # 16-byte key iv = os.urandom(16) # random IV per message cipher = AES.new(key, AES.MODE_CBC, iv) ct = cipher.encrypt(pad(plaintext.encode(), 16)) return base64.b64encode(ct), base64.b64encode(iv) def aes_decrypt(ct_b64, iv_b64, key_str): key = hashlib.sha256(key_str.encode()).digest()[:16] cipher = AES.new(key, AES.MODE_CBC, base64.b64decode(iv_b64)) return unpad(cipher.decrypt(base64.b64decode(ct_b64)), 16).decode()

🏗️ Architecture Overview

This is a Client-Server Application. The browser (client) sends HTTP requests to the Flask server (backend). All cryptographic operations happen on the server — the client only displays results.

Flask (Python) SHA-256 — Practical 9 AES-CBC — Practical 5 Session Auth

🔏 1. SHA-256 Password Hashing (Practical 9)

Passwords are never stored in plain text. On registration, the password is hashed using SHA-256. On login, the entered password is hashed and compared to the stored hash.

import hashlib def sha256_hash(text: str) -> str: # SHA-256 produces a fixed 256-bit (64 hex char) digest return hashlib.sha256(text.encode()).hexdigest() # Registration: password_hash = sha256_hash("user_password") users_db[username] = { "password_hash": password_hash } # Login — compare hashes, never plain passwords: if sha256_hash(entered_pass) == users_db[username]["password_hash"]: # ✓ Access granted

🔒 2. AES Encryption — CBC Mode (Practical 5)

AES (Advanced Encryption Standard) uses a 128-bit key. The user's key string is hashed to produce exactly 16 bytes. A random IV (Initialization Vector) is generated for each encryption to ensure the same plaintext produces different ciphertext every time.

from Crypto.Cipher import AES from Crypto.Util.Padding import pad, unpad import os, base64, hashlib def aes_encrypt(plaintext, key_str): key = hashlib.sha256(key_str.encode()).digest()[:16] # 16-byte key iv = os.urandom(16) # Random IV cipher = AES.new(key, AES.MODE_CBC, iv) # CBC mode ct = cipher.encrypt(pad(plaintext.encode(), 16)) # Pad + encrypt return base64.b64encode(ct), base64.b64encode(iv) def aes_decrypt(ciphertext_b64, iv_b64, key_str): key = hashlib.sha256(key_str.encode()).digest()[:16] ct = base64.b64decode(ciphertext_b64) iv = base64.b64decode(iv_b64) cipher = AES.new(key, AES.MODE_CBC, iv) return unpad(cipher.decrypt(ct), 16).decode()

🔑 3. Authentication (Session-based)

After successful login (hash comparison), Flask creates a server-side session. The encrypt/decrypt endpoints check for this session. If not logged in, they return HTTP 401 Unauthorized.

# Login sets the session session["username"] = username # Protected endpoint checks session if "username" not in session: return jsonify({"message": "Please login first."}), 401 # Logout clears the session session.pop("username", None)

🌐 4. Client-Server Flow (HTTP)

The browser communicates with Flask using JSON over HTTP. Each button click sends a fetch() request to the appropriate API endpoint.

// Client-side JavaScript const response = await fetch("/api/login", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ username, password }) }); const data = await response.json();

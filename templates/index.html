Diffie-Hellman Key Exchange Practical 7
Shared AES key derived — never transmitted
DH_P = # 1024-bit RFC 3526 prime DH_G = 2 # generator def dh_keypair(): private = random_int() % (DH_P - 2) + 2 public = pow(DH_G, private, DH_P) # g^private mod p return private, public def dh_shared(their_public, my_private): shared = pow(their_public, my_private, DH_P) # their^mine mod p return hashlib.sha256(shared).digest()[:16] # → 16-byte AES key # Both sides compute same key: # client: pow(B, a, p) == server: pow(A, b, p)

def aes_encrypt(plaintext, key): # key = DH shared secret iv = os.urandom(16) cipher = AES.new(key, AES.MODE_CBC, iv) ct = cipher.encrypt(pad(plaintext.encode(), 16)) return b64(ct), b64(iv) # HMAC-SHA256 integrity tag tag = hmac.new(key, (ciphertext + iv).encode(), sha256).hexdigest() # On decrypt — verify HMAC FIRST, then decrypt if not hmac.compare_digest(expected_tag, received_tag): return 400 # reject tampered data

Rainbow Table Attack SHA-256 alone — attacker looks up hash in pre-computed table SHA-256 + random 32-byte salt (Practical 9) — each hash is unique
Key Interception AES key sent in request body — interceptable Diffie-Hellman (Practical 7) — key derived, never transmitted
Brute Force Login Unlimited attempts — bot can try millions of passwords Rate limiting: 5 attempts → 60s lockout per IP
Message Tampering No integrity check — attacker can modify ciphertext bits HMAC-SHA256 tag — tampered messages rejected before decryption
Session Hijacking Weak session cookies — no expiry, no secure flags HttpOnly + Secure + SameSite=Strict + 30min expiry
Timing Attack String comparison (==) leaks timing info about hash hmac.compare_digest() — constant-time comparison
